2. Configuration

2.1 SMO: VES Collector

Edit configuration of VES Collector:



# Edit the configuration
kubectl edit configmap -n onap onap-dcae-ves-collector-application-config-configmap
  • auth.method: certBasicAuth -> noAuth
  • collector.service.port: 8080 -> 8443

Restart the VES Collector:






# Find the pod
kubectl get pod -n onap | grep ves

# Delete the pod, it will restart one automatically
kubectl delete pod -n onap <name_of_vescollector_pod>

3.1 Raise Alarm

Expected Flowchart:

Raise Alarm Using REST API:

  • Environment Variable: 
    


    ​​​​export Service_alarmmanager=$(kubectl get services -n ricplt | grep "\-alarmmanager-http" | cut -f1 -d ' ')
​​​​export Alarmmanager_IP=$(kubectl get svc ${Service_alarmmanager} -n ricplt -o yaml | grep clusterIP | awk '{print $2}')
​​​​echo ${Alarmmanager_IP}
  • Define Alarm: 
    ​​​​curl --location --request POST "http://${Alarmmanager_IP}:8080/ric/v1/alarms/define" -H 'accept: application/json' -H 'Content-Type: application/json' --data-raw '{"alarmdefinitions": [{"alarmId":9002,"alarmText":"HW-1","eventtype": "Communication error", "operationinstructions": "Not defined"}]}'
  • Raise Alarm: 
    ​​​​curl --location --request POST "http://${Alarmmanager_IP}:8080/ric/v1/alarms" -H 'accept: application/json' -H 'Content-Type: application/json' --data-raw '{"managedObjectId": "RIC","applicationId": "HW-1","specificProblem": 9002,"perceivedSeverity": "CRITICAL","identifyingInfo": "INFO-1","AlarmAction": "RAISE"}'
