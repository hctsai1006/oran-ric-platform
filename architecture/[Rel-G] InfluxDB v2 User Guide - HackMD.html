1. Introduction

1.1 Releationship

InfluxDB is a database server, influx CLI is a client. We can create several configs using the influx CLI, which enables the user to connect to several influxDB using a single influx CLI.

Create Config:
Create a configuration for influxDB server in the influx cli.






influx config create --config-name admin \
  --host-url http://localhost:8086 \
  --org influxdata \
  --token  Eki592tJqjTB47A3KHBpbFpfIoecPVjd\
  --active

List User:

influx user list

Reset Password for Admin:

influx user password --name admin --password 7jQCNdujbSKju7cL32IzOOwAx7rEjEGJ

5. Dashboard

5.1 Variable

Reference:

  • Variable types
  • Query Tag
  • Common variable queries

Create Variable from Tag:









import "influxdata/influxdb/schema"

schema.tagValues(
    bucket: "kpimon",
    tag: "Viavi_UE_id",
    start: -1h,
    stop: now()
)
  • Note: Schema
    This is the schema for schema.tagValues. We can find that there are some ? before the name, it means the optional value. 
    
















    ​​​​(
​​​​    bucket: string,
​​​​    tag: string,
​​​​    ?predicate: (
​​​​        r: {
​​​​            A with
​​​​            _value: B,
​​​​            _time: time,
​​​​            _stop: time,
​​​​            _start: time,
​​​​            _measurement: string,
​​​​            _field: string,
​​​​        },
​​​​    ) => bool,
​​​​    ?start: C,
​​​​    ?stop: D,
​​​​) => stream[E] where E: Record

Create Variable from Variable:

  • Note: Do remember to rename the column to _value








from(bucket: "kpimon")
  |> range(start: v.timeRangeStart, stop: v.timeRangeStop)
  |> filter(fn: (r) => r["_measurement"] == "UeMetrics")
  |> filter(fn: (r) => r["Viavi_UE_id"] == v.UeId)
  |> group(columns: ["Viavi_Slice_id"])
  |> distinct(column: "Viavi_Slice_id")
  |> keep(columns: ["Viavi_Slice_id"])
  |> rename(columns: {Viavi_Slice_id: "_value"})
